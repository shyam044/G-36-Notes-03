FastAPI – User & Book API

Concept

This FastAPI app demonstrates how to use Pydantic models to structure APIs for handling user data (with nested models) and book records.

Python Code

from fastapi import FastAPI, Response
from pydantic import BaseModel

class User(BaseModel):
    uname: str
    email: str
    password: str

class Profile(BaseModel):
    college: str
    address: str

class UserPublic(BaseModel):
    uname: str
    email: str
    profile: Profile

class BookSchema(BaseModel):
    name: str
    price: float

app = FastAPI()

@app.get("/")
def wish():
    return "Hii User!!"

books = {}

@app.post("/add_book")
def add_book(book: BookSchema):
    if book.name not in books:
        books[book.name] = book.price
    return Response("Successfully added the book to the db!!", 200)

@app.get("/get_books", response_model=dict)
def get_books():
    return books

users = {
    'uname': 'Bhargav',
    "email": "Bhargav@gmail.com",
    "password": "1234",
    "profile": {
        "college": "Reva Univer",
        "address": "Bengaluru"
    }
}

@app.get("/user/", response_model=User)
def get_user():
    return users

if __name__ == "__main__":
    import uvicorn
    uvicorn.run(app)

Explanation

  • BaseModel: All data models like User, Profile, and BookSchema inherit from it for validation.
  • /add_book: Adds book data to an in-memory dictionary if the book name is unique.
  • /get_books: Returns all books currently added.
  • /user/: Returns a static user (with password shown here, but should be hidden in real apps).
  • Nested Model: The Profile model is embedded inside UserPublic.
Note:
  • To **hide sensitive fields** like passwords, use UserPublic instead of User in the response_model.
  • This demo uses in-memory Python dictionaries. Use a database in production apps.
