FastAPI – User & Book API

Concept

This FastAPI app demonstrates how to use Pydantic models to create structured APIs for managing user data (with nested models) and book records.

Python Code

from fastapi import FastAPI, Response
from pydantic import BaseModel

class User(BaseModel):
    uname : str
    email : str
    password : str

class Profile(BaseModel):
    college : str
    address : str

class UserPublic(BaseModel):
    uname : str
    email : str
    profile : Profile

class BookSchema(BaseModel):
    name : str
    price : float

app = FastAPI()

@app.get("/")
def wish():
    return "Hii User!!"

books = {}
@app.post("/add_book")
def add_book(book : BookSchema):
    if book.name not in books:
        books[book.name] = book.price
    return Response("Successfully added the book to the db!!", 200)

@app.get("/get_books", response_model=dict)
def get_books():
    return books

users = {
    'uname' : 'Bhargav',
    "email" : "Bhargav@gmail.com",
    "password" : "1234",
    "profile" : {
        "college" : "Reva Univer",
        "address" : "Bengaluru"
    }
}

@app.get("/user/", response_model=User)
def get_user():
    return users

if __name__ == "__main__":
    import uvicorn
    uvicorn.run(app)

Sample Outputs

GET /
→ "Hii User!!"

POST /add_book
Body: {"name": "Python 101", "price": 349.0}
→ "Successfully added the book to the db!!"

GET /get_books
→ {"Python 101": 349.0}

GET /user/
→ {
  "uname": "Bhargav",
  "email": "Bhargav@gmail.com",
  "password": "1234"
}

Explanation

  • BaseModel: All data models (User, Profile, Book) inherit from this for validation.
  • add_book: Accepts a book and adds it to an in-memory dictionary.
  • get_books: Returns all added books.
  • get_user: Returns a sample user. Notice that password is exposed (in real apps, you'd avoid this).
  • Nested Models: The Profile model is nested inside UserPublic.
Note:
  • Use UserPublic instead of User in response_model if you want to hide sensitive data like password.
  • This example uses in-memory storage. In production, use a database.
